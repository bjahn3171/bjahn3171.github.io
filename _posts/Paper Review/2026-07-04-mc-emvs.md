## Abstract / 요약

### 1. Original Abstract

<details markdown="1">
<summary><strong>Original Abstract 보기</strong></summary>

> Event cameras are bio-inspired sensors that offer advantages over traditional cameras. They operate asynchronously,
sampling the scene at microsecond resolution and producing a stream of brightness changes. This unconventional output has sparked
novel computer vision methods to unlock the camera’s potential. Here, the problem of event-based stereo 3D reconstruction for SLAM
is considered. Most event-based stereo methods attempt to exploit the high temporal resolution of the camera and the simultaneity of
events across cameras to establish matches and estimate depth. By contrast, this work investigates how to estimate depth without
explicit data association by fusing Disparity Space Images (DSIs) originated in efficient monocular methods. Fusion theory is
developed and applied to design multi-camera 3D reconstruction algorithms that produce state-of-the-art results, as confirmed by
comparisons with four baseline methods and tests on a variety of available datasets.
</details>

### 2. 한국어 요약

본 논문은 SLAM을 위한 이벤트 기반 스테레오 3차원 재구성 문제를 다룬다. 기존의 대부분의 이벤트 기반 스테레오 방법은 이벤트 카메라의 높은 시간 해상도와 다중 카메라에서 발생하는 이벤트 간 동시성을 활용하여 대응 관계를 찾고, 이를 바탕으로 깊이를 추정한다.
반면 본 연구는 명시적인 이벤트 매칭 과정 없이 깊이를 추정하는 방법을 제안한다. 이를 위해 단안 방식으로 생성된 Disparity Space Image(DSI)를 융합하여 스테레오 깊이 정보를 구성한다.
또한 본 논문은 DSI 융합에 대한 이론적 기반을 제시하고, 이를 다중 카메라 기반 3차원 재구성 알고리즘 설계에 적용한다. 제안 방법은 네 가지 baseline 방법과의 비교 및 다양한 공개 데이터셋에서의 실험을 통해 기존 방법 대비 우수한 성능을 보임을 확인하였다.


## 서론

본 논문은 VO/SLAM을 위한 이벤트 기반 스테레오 3차원 재구성 문제를 다룬다. 이벤트 카메라는 밝기 변화가 발생한 순간에만 이벤트를 비동기적으로 출력하는 뉴로모픽 센서로, 높은 시간 해상도와 넓은 동적 범위, 낮은 전력 소비라는 장점을 가진다. 
이러한 특성은 고속 움직임이나 극한 조명 환경에서 유용하지만, 이벤트 스트림을 처리하기 위한 별도의 알고리즘이 필요하다.

본 연구는 이벤트 기반 단안 VO 시스템인 EVO와 그 핵심 모듈인 EMVS에서 출발한다. EMVS는 이미지 복원이나 명시적인 이벤트 매칭 없이도 효율적인 3차원 재구성이 가능하다는 장점을 가진다.
논문은 이를 공통 clock을 공유하는 다중 이벤트 카메라, 특히 스테레오 설정으로 확장하고, 재초점화된 이벤트 정보를 융합하기 위한 이론을 제시한다. 이를 통해 단안 방식보다 더 정확한 깊이 추정과 이상치 제거가 가능함을 보인다.


## 배경

### 1. 이벤트 카메라 모델

Dynamic Vision Sensor(DVS)와 같은 이벤트 카메라는 각 픽셀이 독립적으로 밝기 변화를 감지한다. 픽셀 $\mathbf{x}_k$에서 이전 이벤트 시점 $t_k - \Delta t_k$ 대비 현재 시점 $t_k$의 로그 밝기 변화가 contrast threshold $\theta$를 넘으면 이벤트가 발생한다.

$$
L(\mathbf{x}_k, t_k) - L(\mathbf{x}_k, t_k - \Delta t_k) = p_k \theta
$$

여기서 $L$은 로그 밝기, $p_k$는 polarity이다. 로그 밝기가 증가하면 positive event, 감소하면 negative event가 발생한다.

이벤트 카메라의 중요한 특징은 다음과 같다.

- 모든 픽셀이 같은 frame rate를 갖지 않는다.
- 변화가 있는 픽셀만 이벤트를 발생시킨다.
- motion과 texture가 있는 영역에서 이벤트가 많이 발생한다.
- 장면의 edge나 high-gradient 구조가 sparse하게 관측된다.

이 sparse edge-like output은 한편으로는 어려움이지만, 다른 한편으로는 3D reconstruction에 유리한 단서가 된다. 실제 3D edge에서 발생한 이벤트 ray들은 올바른 depth 위치에서 반복적으로 교차하기 때문이다.


$$
L(\mathbf{x}_k, t_k) - L(\mathbf{x}_k, t_k - \Delta t_k) = p_k \theta
$$

즉, 로그 밝기가 충분히 증가하면 positive event가, 충분히 감소하면 negative event가 발생한다. 이처럼 이벤트 카메라는 모든 픽셀을 고정된 frame rate로 샘플링하지 않고, 각 픽셀이 독립적인 sampling rate를 갖는다.

조명이 일정하다고 가정하면 이벤트는 주로 장면의 움직임과 texture가 있는 위치에서 발생한다. 따라서 이벤트 스트림은 장면 전체가 아니라 변화가 있는 구조적 영역을 중심으로 생성된다. EMVS는 이러한 이벤트의 특성을 이용해 3차원 구조를 추정한다.

### 2. 기존 event-based stereo depth estimation

기존 이벤트 기반 stereo 방법은 대체로 event simultaneity를 이용한다. 즉, 같은 3D point 또는 edge가 왼쪽 카메라와 오른쪽 카메라에서 거의 같은 시간에 이벤트를 발생시킨다고 가정한다. 이 가정을 이용하면 epipolar line 위에서 timestamp가 비슷한 이벤트를 correspondence candidate로 볼 수 있다.

하지만 이 접근은 다음 문제가 있다.

1. **이벤트 하나의 정보량이 작다.**  
   이벤트는 위치, 시간, polarity만 가지므로 patch texture가 풍부한 frame image에 비해 matching cue가 부족하다.

2. **event timing은 완전히 동시적이지 않다.**  
   카메라 threshold, local contrast, viewing angle, noise, motion direction에 따라 이벤트 발생 시점이 달라질 수 있다.

3. **event appearance가 motion에 의존한다.**  
   같은 물체 edge라도 움직임 방향에 따라 이벤트 분포가 달라질 수 있다.

4. **short-term stereo는 parallax가 부족할 수 있다.**  
   짧은 시간 구간의 이벤트만 사용하면 depth를 안정적으로 추정하기 어렵다.

ESVO와 같은 방법은 time surface와 probabilistic filtering을 사용해 이러한 문제를 완화하지만, 여전히 stereo matching 또는 local consistency 기반의 data association을 필요로 한다.

### 3. EMVS: 단안 이벤트 기반 3D 재구성

EMVS(Event-based Multi-View Stereo)는 단안 이벤트 카메라와 카메라 pose가 주어졌을 때 3D 구조를 추정하는 방법이다. 핵심은 space sweeping을 이용해 이벤트를 3D 공간으로 back-project하고, ray density가 높은 위치를 찾는 것이다.

EMVS의 절차는 다음과 같다.

1. reference view를 정한다.
2. 가능한 depth plane들을 정의한다.
3. 각 이벤트를 해당 이벤트 시점의 camera pose를 이용해 여러 depth plane으로 warp한다.
4. warp된 이벤트를 DSI에 누적한다.
5. 각 reference pixel에서 depth 방향으로 DSI 값을 탐색한다.
6. DSI 값이 최대가 되는 depth를 해당 픽셀의 depth로 선택한다.

DSI는 다음과 같이 직관적으로 이해할 수 있다.

$$
\mathrm{DSI}(X)
=
\text{3D voxel } X \text{를 지나가는 event ray의 개수}
$$

실제 3D edge가 있는 위치에서는 서로 다른 viewpoint에서 온 ray들이 모이므로 DSI 값이 커진다. 따라서 DSI의 local maximum은 실제 장면 구조의 후보가 된다.

EMVS의 장점은 다음과 같다.

- image reconstruction 없이 depth를 추정할 수 있다.
- 이벤트 간 명시적 correspondence가 필요 없다.
- GPU 없이 CPU에서도 효율적으로 동작한다.
- event refocusing 또는 contrast maximization 관점으로 해석할 수 있다.

본 논문은 이 EMVS의 장점을 stereo/multi-camera setting으로 확장한다.

## Method

본 논문은 EMVS의 DSI 기반 구조를 stereo 및 multi-event-camera setting으로 확장한다. 입력은 동기화되고 강체로 고정된 두 개 이상의 이벤트 카메라, 각 카메라의 event stream, camera trajectory, intrinsic/extrinsic calibration이다. 출력은 reference view 기준의 semi-dense depth map과 confidence map이다.

전체 과정은 다음과 같다.

1. 모든 카메라가 공유하는 공통 reference view를 정의한다.
2. 각 카메라의 이벤트를 3D 공간으로 back-project하여 카메라별 DSI를 만든다.
3. 카메라별 DSI를 voxel-wise fusion function으로 융합한다.
4. fused DSI에서 depth 방향 local maximum을 찾는다.
5. confidence thresholding과 filtering을 거쳐 semi-dense depth map을 얻는다.

### 1. 공통 Reference View와 DSI 생성

단순한 방법은 왼쪽 카메라와 오른쪽 카메라에 대해 EMVS를 각각 수행한 뒤, 결과 point cloud를 나중에 합치는 것이다. 하지만 이는 late fusion 방식이므로 두 카메라가 같은 장면을 관측한다는 정보를 충분히 활용하지 못한다.

본 논문은 DSI 단계에서 early fusion을 수행한다. 이를 위해 왼쪽/오른쪽 카메라의 DSI를 서로 다른 좌표계에 만들지 않고, 하나의 공통 reference view에 정의한다. 예를 들어 시간 구간 $[0, T]$의 중간 시점인 $t = T/2$에서의 왼쪽 카메라 pose를 reference view로 사용할 수 있다.

각 카메라별 DSI는 다음처럼 해석할 수 있다.

$$
\mathrm{DSI}_L(X)
=
\text{왼쪽 카메라 이벤트 ray들이 voxel } X \text{를 지나간 횟수}
$$

$$
\mathrm{DSI}_R(X)
=
\text{오른쪽 카메라 이벤트 ray들이 voxel } X \text{를 지나간 횟수}
$$

두 DSI가 같은 voxel grid 위에 정의되므로, 이후 voxel 단위로 직접 비교하고 융합할 수 있다.

### 2. 카메라 간 DSI Fusion

가장 단순한 fusion은 두 카메라의 이벤트 ray를 하나의 DSI에 모두 누적하는 것이다. 그러나 이 경우 특정 voxel의 ray count가 높더라도, 그 ray들이 양쪽 카메라에서 균형 있게 온 것인지 한쪽 카메라에서만 온 것인지 알 수 없다.

실제 3D edge라면 두 카메라 모두에서 ray density가 높게 나타날 가능성이 크다. 반대로 한쪽 카메라에서만 값이 높다면 noise, occlusion, accidental ray intersection일 수 있다. 따라서 좋은 fusion function은 다음 조건을 만족해야 한다.

> 두 카메라의 DSI 값이 모두 높을 때만 높은 score를 준다.

논문에서 핵심적으로 사용하는 fusion function은 harmonic mean이다.

$$
H(u, v)
=
\frac{2uv}{u + v}
$$

여기서 $u = \mathrm{DSI}_L(X)$, $v = \mathrm{DSI}_R(X)$이다.

Harmonic mean은 한쪽 값만 클 때 결과값을 강하게 낮춘다. 따라서 양쪽 카메라에서 모두 지지되는 voxel을 강조하고, 한쪽에서만 발생한 spurious ray intersection을 억제한다.

### 3. Fusion Function 비교

논문은 다음과 같은 fusion function을 비교한다.

$$
\min(u,v)
\le
H(u,v)
\le
G(u,v)
\le
A(u,v)
\le
\mathrm{RMS}(u,v)
\le
\max(u,v)
$$

여기서 $A$는 arithmetic mean, $G$는 geometric mean, $H$는 harmonic mean이다.

- `max`, `RMS`는 한쪽 카메라에서만 값이 높아도 큰 score를 줄 수 있다.
- `A`는 두 카메라 ray count를 단순 평균내는 방식에 가깝다.
- `G`, `H`, `min`은 두 카메라에서 모두 높은 값이 나와야 큰 score를 준다.
- `min`은 너무 보수적으로 정보를 잘라낼 수 있다.
- `H`는 한쪽만 큰 경우를 강하게 억제하면서도 두 입력 정보를 부드럽게 반영한다.

따라서 본 논문에서는 harmonic mean이 camera fusion에 적합한 선택으로 사용된다.

### 4. Depth Map과 Confidence Map 추출

DSI fusion이 끝나면 fused DSI에서 depth를 추출한다. fused DSI를 $f(X)$라고 할 때, 각 reference pixel $(x, y)$에 대해 depth 방향으로 값을 탐색한다.

$$
Z^*(x, y)
=
\operatorname*{arg\,max}_{Z}
f(X(x, y, Z))
$$

해당 maximum value는 confidence로 사용된다.

$$
c^*(x, y)
=
\max_Z f(X(x, y, Z))
$$

즉, depth map은 DSI가 가장 잘 refocus되는 depth이고, confidence map은 그 refocusing score의 크기이다. 이후 Adaptive Gaussian Thresholding과 median filtering을 적용하여 신뢰도 높은 pixel만 남긴다. 최종 출력은 dense depth map이 아니라 edge 중심의 semi-dense depth map이다.

### 5. Temporal Fusion과 Event Simultaneity

본 논문은 카메라 간 fusion뿐 아니라 시간 구간 간 fusion도 다룬다. 전체 이벤트 구간을 여러 sub-interval로 나누고, 각 sub-interval마다 DSI를 만든 뒤 다시 융합한다.

예를 들어 stereo setting에서 시간 구간을 $N_s$개로 나누면 총 $2N_s$개의 DSI가 생성된다. 이후 같은 시간 구간의 left/right DSI를 먼저 harmonic mean으로 융합하고, 여러 시간 구간의 결과를 arithmetic mean으로 합칠 수 있다.

흥미로운 점은 이 방법이 event simultaneity에 강하게 의존하지 않는다는 것이다. 논문은 left/right sub-interval을 일부러 shuffle하여 서로 다른 시간 구간의 DSI를 fusion하는 실험을 수행한다. 결과는 약간 나빠지지만 완전히 붕괴하지 않는다. 이는 이 방법에서 stereo matching이 event-to-event level이 아니라 DSI-to-DSI level에서 이루어진다는 것을 보여준다.


## Experiments / Analysis

### 1. 실험 구성

논문은 여러 데이터셋을 사용하지만, 리뷰에서는 핵심 실험만 정리한다.

| 실험 | 목적 |
|---|---|
| Fusion function 비교 | 어떤 DSI fusion이 outlier rejection에 좋은지 확인 |
| MVSEC 비교 | 기존 event-based stereo method와 정량 비교 |
| Temporal fusion / shuffling | temporal fusion 효과와 event simultaneity 필요성 분석 |
| DSEC driving | 실제 driving scenario에서 성능 확인 |

정량 평가는 주로 depth error와 bad-pix를 본다.

| Metric | 의미 |
|---|---|
| Mean Abs Error | 예측 depth와 GT depth의 평균 절대 오차 |
| Median Abs Error | outlier에 덜 민감한 절대 오차 중앙값 |
| bad-pix | 오차가 일정 기준 이상인 pixel 비율 |
| #Points | reconstruction된 point 수 |

### 2. Fusion Function 비교

먼저 논문은 `min`, `H`, `G`, `A`, `RMS`, `max`를 비교한다. 결과적으로 `H`와 `min`처럼 arithmetic mean보다 보수적인 함수가 outlier를 더 잘 억제한다.

특히 harmonic mean은 두 카메라에서 모두 높은 ray density가 나타나는 voxel을 강조한다. 반대로 `max`나 `RMS`는 한쪽 카메라에서만 높은 값이 나와도 큰 score를 줄 수 있기 때문에 outlier가 더 많이 남는다.

정리하면 다음과 같다.

| Fusion | 해석 |
|---|---|
| `max`, `RMS` | 한쪽 카메라 evidence만으로도 강한 응답을 줄 수 있어 outlier에 취약 |
| `A` | 단순 ray count 평균에 가까움 |
| `G`, `H`, `min` | 두 카메라 모두에서 지지되는 구조를 강조 |
| `H` | accuracy와 reconstruction density 사이의 균형이 좋음 |

따라서 본 논문에서는 harmonic mean을 기본적인 camera fusion function으로 사용한다.

### 3. MVSEC에서 기존 방법과 비교

<figure class="paper-figure paper-figure--wide">
  <img src="{{ '/assets/images/paper_review/mc-emvs/eval visual.png' | relative_url }}" alt="MCEVMS overview">
</figure>

MVSEC indoor flying sequence에서 제안 방법은 EMVS, ESVO, SGM, GTS와 비교된다. 핵심 결과만 보면 다음과 같다.

| Method | Mean Err [cm] ↓ | Median Err [cm] ↓ | bad-pix [%] ↓ | #Points [M] ↑ |
|---|---:|---:|---:|---:|
| EMVS (monocular) | 33.78 | 14.35 | 3.84 | 1.27 |
| ESVO | 25.00 | 10.59 | 3.35 | 2.04 |
| ESVO indep. 1s | 22.70 | 9.83 | 2.83 | 1.56 |
| SGM indep. 1s | 35.42 | 12.35 | 6.39 | 14.46 |
| GTS indep. 1s | 389.00 | 45.43 | 38.45 | 0.06 |
| Ours: $H_c \circ A_t$ | **20.07** | **9.53** | **1.35** | 0.81 |
| Ours: $H_c \circ A_t$ + MF | 20.64 | 9.72 | 1.43 | 3.00 |

제안 방법은 EMVS보다 낮은 error를 보이며, ESVO보다도 mean error, median error, bad-pix에서 좋은 결과를 낸다. 특히 bad-pix가 크게 줄어드는 점이 중요하다. 이는 DSI fusion이 단순히 depth를 더 많이 찍는 것이 아니라 outlier를 걸러내는 데 효과적임을 보여준다.

다만 제안 방법의 point 수는 ESVO보다 적다. 논문은 이를 edge가 더 얇고 정확하게 reconstruction되기 때문으로 해석한다. Morphological filter를 적용하면 point 수가 0.81M에서 3.00M으로 증가하면서도 accuracy 저하는 크지 않다.

### 4. Temporal Fusion과 Shuffling

<figure class="paper-figure paper-figure--wide">
  <img src="{{ '/assets/images/paper_review/mc-emvs/temporal fusion.png' | relative_url }}" alt="MCEVMS overview">
</figure>

Temporal fusion에서는 시간 구간을 여러 sub-interval로 나누고, 각 구간의 DSI를 다시 융합한다. $N_s$를 2, 4, 8로 늘리면 다음과 같이 성능이 조금씩 개선된다.

| $N_s$ | Mean Err [cm] ↓ | Median Err [cm] ↓ | bad-pix [%] ↓ | #Points [M] ↑ |
|---:|---:|---:|---:|---:|
| 2 | 20.92 | 9.76 | 1.66 | 1.14 |
| 4 | 20.49 | 9.58 | 1.63 | 1.19 |
| 8 | **19.80** | **9.27** | **1.56** | **1.20** |

Temporal fusion도 도움이 되지만, monocular에서 stereo로 확장했을 때의 개선 폭이 더 크다. 즉, 시간 축 fusion보다 추가 카메라가 제공하는 stereo parallax가 더 중요한 역할을 한다.

Shuffling 실험도 중요하다. 서로 대응되지 않는 시간 구간의 left/right DSI를 fusion해도 결과가 완전히 무너지지 않는다.

| Method | Mean Err [cm] ↓ | Median Err [cm] ↓ | bad-pix [%] ↓ | #Points [M] ↑ |
|---|---:|---:|---:|---:|
| $A_t \circ H_c$ | 20.92 | 9.76 | 1.66 | 1.14 |
| $A_t \circ H_c$ + shuffling | 22.60 | 10.71 | 1.68 | 1.24 |

이는 제안 방법이 event simultaneity를 필수적으로 요구하지 않는다는 것을 보여준다. 즉, 이 논문에서 stereo는 event matching 문제가 아니라 pose-aligned DSI fusion 문제에 가깝다.

### 5. DSEC Driving Dataset 결과

DSEC는 실제 driving scenario를 포함하는 데이터셋이다. MVSEC outdoor sequence보다 stereo baseline이 크기 때문에 driving depth reconstruction에 더 적합하다.

| Method | Mean Err [m] ↓ | Median Err [m] ↓ | bad-pix [%] ↓ | #Points [M] ↑ |
|---|---:|---:|---:|---:|
| EMVS (monocular) | 5.64 | 2.52 | 13.68 | 1.31 |
| ESVO | 3.88 | 1.56 | 12.08 | 3.40 |
| Ours: $H_c \circ A_t$ | **3.27** | **0.90** | **10.75** | 1.25 |
| Ours: $H_c \circ A_t$ + MF | 3.51 | 0.96 | 11.81 | 3.83 |

DSEC에서도 제안 방법은 EMVS와 ESVO보다 낮은 mean/median error를 보인다. 특히 median error가 크게 낮아진 점은 outlier가 줄고 안정적인 depth가 추정되었음을 의미한다.

다만 driving scene에서는 forward motion이 많기 때문에 image center 부근에서는 apparent motion이 작고 이벤트가 적게 발생할 수 있다. 따라서 주변부보다 중앙부 reconstruction이 어려울 수 있다.


## 6. Discussion

### 6.1 장점

1. 이미지 압축 전체 과정을 end-to-end로 최적화한다.
2. 양자화 문제를 연속적인 relaxation으로 다루어 학습 가능하게 만든다.
3. GDN 비선형성을 사용해 압축에 적합한 representation을 학습한다.
4. JPEG, JPEG 2000과 비교하여 우수한 rate–distortion 성능을 보인다.

### 6.2 한계 및 의문점

1. entropy model이 이후 neural compression 연구들에 비해 상대적으로 단순하다.
2. 실제 codec 시스템 관점에서는 entropy coding, runtime, model size까지 함께 고려할 필요가 있다.
3. 학습 데이터 분포와 다른 이미지에 대해 얼마나 일반화되는지는 추가적으로 볼 필요가 있다.




## 용어 정리

| 용어 | 의미 |
|---|---|
| DVS | Dynamic Vision Sensor, 대표적인 event camera 종류 |
| DSI | Disparity Space Image, depth/disparity 후보 공간에서 event ray density를 누적한 volume |
| Space Sweeping | 여러 depth plane을 가정하고 이벤트 또는 이미지를 warp하여 depth를 찾는 방식 |
| Reference View | DSI를 정의하는 기준 카메라 view |
| Ray Density | 특정 3D voxel을 통과하는 back-projected event ray의 개수 |
| Refocused Events | 특정 depth hypothesis에서 정렬되도록 warp된 이벤트 |
| AGT | Adaptive Gaussian Thresholding, confidence map에서 신뢰도 높은 pixel을 선택하는 thresholding 방법 |
| Semi-dense Depth Map | 전체 pixel이 아니라 edge 중심의 일부 pixel에 대해서만 depth가 있는 map |
| Confidence Map | 각 reference pixel에서 DSI maximum 값을 저장한 map |
| bad-pix | depth error가 기준 이상인 잘못된 pixel 비율 |
| SILog Error | scale-invariant log depth error |
| $\delta$-accuracy | 예측 depth와 GT depth의 비율이 특정 threshold 안에 들어오는 비율 |
